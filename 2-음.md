더 정리하면 

모델1 : DB ↔ 모델 [DAO ↔ DTO] ↔ JSP (컨트롤러 + 뷰)

모델2 : DB ↔ 모델 [DAO ↔ DTO] ↔ Servlet [컨트롤러( + 서비스) ] ↔ 뷰

스프링 : DB ↔ 모델 [DAO(Repository) ↔ Service ↔ DTO] ↔ 컨트롤러 ↔ 뷰

DTO는 모든 계층 사이를 연결

구분,데이터의 형태 (View ↔ Controller),DTO의 주 활동 범위

스프링 (MVC),DTO (커맨드 객체) / Model 객체,View ↔ Controller ↔ Service ↔ DAO
스프링 (REST),DTO (JSON 변환),Client(View) ↔ Controller ↔ Service ↔ DAO

비즈니스 로직 위치,JSP (HTML과 자바 혼재),Controller (Servlet)에 포함,Service 계층으로 완전 독립
DTO 활동 범위,DAO ↔ JSP (단거리),DAO ↔ Service ↔ Controller (중거리),DAO ↔ Service ↔ Controller ↔ View (전 구간)
View ↔ Controller 데이터 형태,(JSP가 곧 컨트롤러라 없음),HTTP 파라미터 (낱개 데이터) 및 Request 객체,DTO 객체 (자동 매핑/커맨드 객체) 또는 JSON
