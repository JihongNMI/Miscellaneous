# 1. 형태 정리

자 여기까지 정리를 좀 해보자.

- SI(System Integrator) = 수탁개발, 도급(청부) 개발 / 상류공정(기획설계) 포함 / 용어상 자사개발도 포함이지만 그건 자사개발로 따로 부르니 생략 /
  >SE = SI하는 사람(상류) / 요구사항, 시스템을 설계, 프로젝트를 관리

- SES(System Engineering Service) = 수탁지원업무, 준위임 / 하류공정(단순 코딩, 테스트, 운용보수) / 상류 할 가능성 그냥 없다고 봄
  >PG = 프로그래머(하류)
  >DBA관련 SES는 대우가 좋다고 하지만 어짜피 얘들도 상류 안하면 거기서 거기일듯

- 시스템개발 = 이름은 그럴 듯하지만 하류공정이잖아? SES도 있고 SI도 있으니 착각 X

- SES는 단순업무, 내지는 그냥 그 업무만 계속이라 커리어 형성 힘듬
  >그렇게 정년까지 할 수 있으면 좋겠지만 그렇게 안되니까 비판이 있겠지 ㅎㅎㅎ

- 자사 > SI > SES
